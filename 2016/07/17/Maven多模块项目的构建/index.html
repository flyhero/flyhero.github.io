Maven多模块项目的构建

原创 2016-07-17 08:55 3 分钟 读完 (约 506 字)

在平时的项目开发中为了便于后期的维护，我们一般会进行分层开发，如果是Java Web的话，最常见的就是分为domain（域模型层）、dao（数据库访问层）、service（业务逻辑层）、web（表现层），这样分层之后，各个层之间的职责会比较明确，后期维护起来也相对比较容易，今天我们就是使用Maven来构建以上的各个层。

创建父项目

打开eclipse New -> project -> maven -> maven Project ->next -> 选择maven-Archetype-site-simple -> 填写项目信息 -> finish
此时父项目创建完毕,项目中生成一个空的src文件夹（可删除）和一个pom.xml文件，pom文件中的 <packaging>pom</packaging>中是pom，这表示pom是可以继承的。所以可以把pom中公共用到的jar包依赖都写在父项目的pom.xml中。

创建子模块

选中父项目，点击右键，选择New -> projec -> maven -> maven Module，点击下一步，在出现的界面中输入子模块的名称，点击下一步，出现Select an Archetype界面。这时选择maven-Archetype-site-quickStart或者maven-Archetype-webapp(构建web层时使用)，然后选择完成，即生成子项目。子模块的<packaging>为jar；web层的<packaging>war</packaging>为war。
