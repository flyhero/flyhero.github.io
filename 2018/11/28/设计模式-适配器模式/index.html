设计模式-适配器模式

原创 2018-11-28 18:15 4 分钟 读完 (约 534 字)

首先要说明的是适配器模式是一种补偿模式，一般设计软件之初首先不会采取这种模式，因为适配器模式主要用于接口不兼容的情况下，这种情况往往是开发后期才会产生的问题。

定义

适配器模式(Adapter Pattern) ：将一个接口转换成客户希望的另一个接口，适配器模式使接口不兼容的那些类可以一起工作，其别名为包装器(Wrapper)。适配器模式既可以作为类结构型模式，也可以作为对象结构型模式。

角色

  • Target : 目标抽象类（或接口）
  • Adapter : 适配器类
  • Adaptee : 适配者类
  • Client : 客户类

总结

优点

  • 解耦
    引入一个适配器类类重用现有的适配者类，无须修改原有代码。

  • 透明性和复用性
    将原有具体实现类封装在适配器类中，对客户比较透明，也提高了适配者的复用。

缺点

  • 类适配器
    继承适配者类，导致无法继承其他类，一次只能适配一个适配者类。

  • 对象适配器
    想要置换适配者类的方法不容易。
